# Contributing to CoreWar KMP

First off, thank you for considering contributing to CoreWar KMP! It's people like you that make the open source community such a great place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
- Use the [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md).
- Describe the expected behavior and the actual behavior.
- Provide a step-by-step reproduction if possible.

### Suggesting Enhancements
- Check [SUGGESTIONS.md](SUGGESTIONS.md) to see if your idea is already listed.
- Use the [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md).

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Issue that pull request!

## Style Guide

### Kotlin
- Follow [Official Kotlin Style Guide](https://kotlinlang.org/docs/coding-conventions.html).
- Use meaningful variable names.
- Keep functions concise.

### Git Commits
- Use descriptive commit messages.
- Reference issues where applicable.

## Branch Naming
- `feature/your-feature-name`
- `fix/bug-description`
- `docs/what-changed`

## Code of Conduct
By contributing, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).
